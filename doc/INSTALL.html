Installing

  • Using LuaRocks
  • Using Git

cdecl for GCC may be installed using LuaRocks or Git.

Before installing the module, make sure to install the Lua plugin for GCC.

Using LuaRocks

Install the module in your home directory:

luarocks install --local gcc-lua-cdecl

Verify that the module has been installed:

luarocks list gcc-lua-cdecl

To use the module, set the Lua module path:

eval `luarocks path`

The command can be added to the shell’s startup file, e.g., ~/.bashrc.

Using Git

Obtain the source from the git repository:

git clone http://git.colberg.org/gcc-lua-cdecl

In the repository, checkout the latest release version:

git checkout 1.2.0

If you have GnuPG, verify that the tag is signed by the following key:

git tag -v 1.2.0
Primary key fingerprint: 2997 6EC2 2BD1 EBBC CC77  F3A6 183B D5ED 3527 8611
     Subkey fingerprint: 983B F9D0 F02A 6014 0CEC  529F 767E 12A8 C92A 31CB

The module is accompanied by a test suite:

make test

By default, the tests assume that the gcc‑lua and gcc‑lua‑cdecl repositories are located in the same parent directory. If the gcc‑lua plugin is installed in the GCC plugin directory instead, override the make variable GCCLUA:

make test GCCLUA=gcclua
